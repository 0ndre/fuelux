Repeater Views

A repeater view is an object with various attributes used to render data within the repeater control. Each view has a unique name, and is selectable via the .repeater-views classed button group. The view object is added to the$.fn.repeater.views object upon load using it's unique view name as a key. Find more information on creating a repeater view extension below, or check out this simple example.

Default Values and Methods

Default values and methods can be added in the same way as general extensions, described above.

Functions

All functions within the repeater view object are called with the repeater instance's context of this. They are passed a helpers object and callback function. The callback function must be ran to proceed to the next step. It can be provided an object as an argument to perform various functions specific to each function, detailed later on.

View Object

The main view object serves as the base for the repeater view extension and supports the following attributes:

Name type description
initialize function Runs only once with each instantiation of the repeater, right at the beginning. Provided a helpers object and callback function.
selected function Runs every time this repeater view is selected prior to rendering. Provided a helpers object and callback function.
renderer object A renderer object, with various attributes specifying how to render content or data. Provided a helpers object and callback function. More details below.

Renderer Object

The renderer object serves to render content or data at a specific level. It has the following structure:

Name type description
after function Runs third in line, after the render function has been called and the content has been appended to the DOM. Provided a helpers object and callback function. The callback accepts an object with the skipNested attribute. If skipNested is set to true, the repeater will ignore rendering this renderer's nested array.
before function Runs first in line, before the render function. Provided a helpers object and callback function. The callback accepts an object with the skipNested attribute. If skipNested is set to true, the repeater will ignore rendering this renderer's nested array.
complete function Runs last, after all the renderers in the nested array have ran. Provided a helpers object and callback function.
nested array Array of renderer objects. These objects have the same structure as the one currently being described. Each one will be ran in order following the current object's render function unless told otherwise.
render function Runs second, after the before function. Primary function for actually rendering data and adding content to the parent container. Provided a helpers object and callback function. The callback function accepts an object with several attributes: item, action, and skipNested. The item attribute represents the desired content to be injected into the DOM, and can be either a string or jQuery object. Item content can have special data-attributes for additional rendering behaviors, listed further below. The action attribute, a string, specifies how the item will be injected in relation to the parent container. By default, it's value is append, but it can be any valid jQuery DOM injection method (ex: 'prepend', 'after', etc). The skipNested attribute, if set to true, will tell the repeater to ignore the current renderer's nested array.
repeat string If set, this renderer will be rendered multiple times over a subset of data. The string value specifies which subset this renderer is to be iterated over. Represents an attribute of an object using dot notation. The root level must be 'data' or 'this'. If the root is 'data', the root object will be the data object returned from the dataSource. If the root is 'this', the root object will be the current repeater context of this. Further levels can be specified, separated by dots. The final level must be an array or iteration will not function correctly. This array will then be passed as the subset attribute in the helpers object for other renderer functions. (ex: 'data.animals.pigeons')

Helpers Object

The helpers object contains various attributes useful for rendering data:

Name type description
container jQuery object The parent container for the current renderer. Default is the .repeater-canvas div. This will remain true unless another container is specified via data-attributes.
data object The data object as returned from the dataSource, in it's entirety.
index number Only present if the repeat attribute was set for the current renderer. Specifies the current index of the subset being iterated over.
item str or jQuery object Represents the item returned from the current renderer's render function. Only present in functions following the render function, and only if an item was returned.
subset array Only present if the repeat attribute was set for the current renderer. Specifies the subset being iterated over.

Data Attributes

There are a few data-attributes that can be added to any markup when rendering content to specify different rendering behaviors. Here are the available attributes:

Name Value(s) description
data-container "true" If this is added to an element within a returned item, the element will become the parent container for any nested renderer within that renderer object.
data-infinite "true" If infinite scrolling is enabled, the element with this attribute will be used as the infinite scrolling container.
data-preserve "deep" or "shallow" If an element contains this attribute it will be preserved upon clearing the repeater of data, which happens each time the dataSource is called. Setting the value to "deep" will tell the repeater to preserve the element and all it's children. Setting the value to "shallow" will tell the repeater to preserve just the element, but will remove any child nodes not marked with data-preserve.
