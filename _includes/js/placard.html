Placards placard.js

Examples

Adds a popup element to inputs/textareas on focus, with additional options for explicit accept/cancel actions.


Header
Cancel

Usage

Via data-attributes

The placard will automatically instantiate any placard with data-initialize="placard". If the control markup is added after page load with data-initialize="placard", this control will be initialized on focus.

Add data-ellipsis="true" to the placard element in order to enable ellipsis for overflow on the input or textarea. NOTE: for textareas, Javascript is used to enable ellipsis. Make sure to use the .('getValue'); and .('setValue'); methods for retrieving/setting value, as it is manipulated upon focus gain/loss. Also, performance drops with larger textareas when ellipsis is enabled.

Via JavaScript

Call the placard control via JavaScript:

{% highlight js %}$('#MyPlacard').placard();{% endhighlight %}

Options

Options can be passed via Javascript upon initialization.

Name type default description
onAccept function undefined Function to be called when the user indicates they want to 'accept' changes. A helpers object containing previousValue and current value is passed as an argument. The default accept behaviors will not occur, leaving it up to the developer to determine what happens next. Useful for validation purposes.
onCancel function undefined Function to be called when the user indicates they want to 'cancel' changes. A helpers object containing previousValue and current value is passed as an argument. The default cancel behaviors will not occur, leaving it up to the developer to determine what happens next. Useful for validation purposes.
externalClickAction string cancel Specifies what action occurs on an external click (click outside the placard element).
externalClickExceptions array [ ] Array of jQuery selector strings. Anything that matches the selector (searched globally) will not be counted as an external click. Useful for allowing other items to be clicked without dismissing the placard.
explicit boolean false Requires the user explicitly select 'accept' or 'cancel' before the placard is dismissed.
revertOnCancel boolean OR number -1 Dictates whether the placard reverts the entered value when a 'cancel' action occurs. -1 checks for a '.placard-accept' element on initialization, setting this value to true if present. Also accepts true or false values.

Methods

$().placard();

Attaches placard to the placard markup.

.placard('getValue');

Gets the current actual placard value.

{% highlight js %}$('#element').placard('getValue');{% endhighlight %}

.placard('hide');

Hides the placard popup.

{% highlight js %}$('#element').placard('hide');{% endhighlight %}

.placard('setValue');

Sets the current actual placard value.

{% highlight js %}$('#element').placard('setValue');{% endhighlight %}
Parameter description
value Required. String or number that the placard value will be set to.

.placard('show');

Shows the placard popup.

{% highlight js %}$('#element').placard('show');{% endhighlight %}

Events

Event Type Description
accept Fired when the user indicates the desire to 'accept' changes. If using the onAccept option, this event fires after that function is called.
cancel Fired when the user indicates the desire to 'cancel' changes. If using the onCancel option, this event fires after that function is called.
hide Fired when the placard is dismissed and the popup dissapears.
show Fired when the placard obtains focus and the popup appears.
