Overview

Individual or compiled

Components can be included individually (using FuelUX's individual *.js files), or all at once (using fuelux.js or the minified fuelux.min.js).

Using the compiled JavaScript

Both fuelux.js and fuelux.min.js contain all plugins in a single file. Include only one.

Plugin dependencies

Some components depend on other components. If you include components individually, make sure to check for these dependencies in the docs below. An error should be thrown if dependencies are not met. Also note that all plugins depend on jQuery and Bootstrap (this means jQuery and Boostrap must be included before the components files). Consult our bower.json to see which versions of jQuery and Bootstrap are supported.

Selector-based initialization

You can initialize all FuelUX components on page load purely through markup without writing a single line of JavaScript by adding its respective class as a descendant of .fuelux.

Alternatively, you can initialize a specific component like this at any time:

{% highlight js %} $('#myElement').selectlist(); {% endhighlight %}

All methods should accept an optional options object, a string which targets a particular method, or nothing (which initiates a plugin with default behavior):

{% highlight js %} $('#myElement').wizard() // initialized with defaults $('#myElement').wizard('selectedItem', { step: 3 });) // initialized with selectedItem $('#myElement').wizard('show') // initializes and invokes show immediately {% endhighlight %}

Each plugin also exposes its raw constructor on a Constructor property: $.fn.popover.Constructor.

Events

FuelUX provides custom events for most components unique actions. Generally, these come in an infinitive and past form - where the infinitive (ex. show) is triggered at the start of an event, and its past form (ex. changed) is trigger on the completion of an action.

As of 3.0.0, all FuelUX events are namespaced.

Third-party libraries

Bootstrap does not officially support third-party JavaScript libraries like Prototype or jQuery UI. Despite .noConflict and namespaced events, there may be compatibility problems that you need to fix on your own.

By default, a dropdown menu is automatically positioned 100% from the top and along the right side of its parent. Remove .dropdown-menu-right to a .dropdown-menu to left align the dropdown menu.

Turn any element into an infinite scrolling region with content that loads on demand.

Markup

Simply place class="infinitescroll" on an element of your choosing (preferably a div or span).

{% highlight html %}
{% endhighlight %}

Options

Options can be passed via Javascript upon initialization.

Name type default description
dataSource function null Called whenever the user scrolls the specified percentage to the bottom. Arguments passed are a helpers object and callback function. The helpers object contains current percentage and scrollTop values. The callback function is used to append more content to the element. Pass an object back within the callback function structured as follows: { content: '...' } If you have no more additonal content to append, add the attribute end: true to that object, which will append '---------' by default and prevent further dataSource calls. Passing a string value for the end attribute will append that string instead of the default.
hybrid boolean OR object false Sets whether to use "hybrid mode," which requires the user to click a button before loading additional content. If set to true, a default "load more" icon will be used within the button. Additionally, it can be set to an object with the following structure: { label: (string or jQuery obj) } The label attribute's value will then be appended within the button instead of the default icon.
percentage number 95 Percentage scrolled to the bottom before the dataSource function is called for more content.

Via data attributes

Add data-ellipsis="true" to the placard element in order to enable ellipsis for overflow on the input or textarea. NOTE: for textareas, Javascript is used to enable ellipsis. Make sure to use the .('getValue'); and .('setValue'); methods for retrieving/setting value, as it is manipulated upon focus gain/loss. Also, performance drops with larger textareas when ellipsis is enabled.

Options

Options can be passed via Javascript upon initialization.

Name type default description

onCancel function undefined Function to be called when the user indicates they want to 'cancel' changes. A helpers object containing previousValue and current value is passed as an argument. The default cancel behaviors will not occur, leaving it up to the developer to determine what happens next. Useful for validation purposes.
externalClickAction string cancel Specifies what action occurs on an external click (click outside the placard element).
externalClickExceptions array [ ] Array of jQuery selector strings. Anything that matches the selector (searched globally) will not be counted as an external click. Useful for allowing other items to be clicked without dismissing the placard.
explicit boolean false Requires the user explicitly select 'accept' or 'cancel' before the placard is dismissed.
revertOnCancel boolean OR number -1 Dictates whether the placard reverts the entered value when a 'cancel' action occurs. -1 checks for a '.placard-accept' element on initialization, setting this value to true if present. Also accepts true or false values.

      Via data attributes

      Add data-staticheight="true" to the repeater element in order to enable the staticHeight option, which will constrain the repeater to a set height dictated by CSS styles with scrollable data. If set to a number instead (ex: data-staticheight="400") the height will be that value in pixels. This option can also be set via Javascript on initialization.

      Options

      Options can be passed via Javascript upon initialization.

      Name type default description
      dataSource function empty function This function is used to provide paging information and data for the repeater. It is called prior to rendering the current view, passing a options object and callback function. The callback function is ran once the appropriate information is gathered to complete the rendering. More details on using dataSource can be found below.
      defaultView string or number -1 Specifies the initial view the repeater will render. This is usually a string value equivalent to the desired repeater view plugin. If set to -1, the repeater will check the .repeater-views button group for an .active class, using it's input value as the defaultView setting.
      dropPagingCap number 10 Specifies the number of items allowed within the .repeater-primaryPaging dropdown menu. If the number of pages exceeds this amount the .repeater-secondaryPaging input will instead be used.
      staticHeight boolean or number -1 Dictates whether the repeater is a set height with scrollable data. If set to true the height will be based off of CSS styles applied to the repeater element. If set to a positive number, that value will be applied as the height in pixels. If set to -1 the repeater will check for the data-staticheight attribute and use its value if present.

      Clears the repeater of current data. Accepts an optional options object as an argument, with the attributes listed in the table below.

      preserve boolean If set to true, rather than simply empty the repeater of data this will scan it for any items with the data-preserve attribute and preserve them while removing all other items. Default is false.

      Data Source

      The dataSource function is called prior to rendering data for the current view. It is passed a options object and callback function as arguments. The options object provides context for which data should be returned, and the callback is used to continue onward with rendering.

      The options object can vary in content depending on which view plugin is being used, though many should share these common attributes. Here is the standard options object format:

      Attribute type description
      filter object Provides context for the selected .repeater-filters dropdown item, representing data filtering. The object contains a text attribute for the displayed text, as well as a value attribute for the selected item's value.
      pageIndex number Represents the current page of data.
      pageSize number Provides context for the selected .repeater-itemization dropdown item value, representing number of data items to be displayed.
      search string Provides context for the entered .repeater-search search box, representing the desired user search value. Only present if a search value has been entered.
      view string Provides context for the selected .repeater-views button group item, representing the current view. Used to determine view-specific return data.

      The dataSource's callback function should be ran after gathering the desired data for rendering. It expects a data object to be passed as an argument. This object's contents will vary depending on the view plugin being used, however here are the common expected attributes:

      Attribute type description
      count number Tells the repeater how many items are being returned in the data, which is used as the .repeater-count element's text value.
      end number Tells the repeater the end index of current displayed data items, which is used as the .repeater-end element's text value.
      page number Sets the current repeater page. Also shown in the .repeater-primaryPaging or .repeater-secondaryPaging element.
      pages number Tells the repeater how many pages of data are available, which is used as the .repeater-pages element's text value.
      start number Tells the repeater the starting index of current displayed data items, which is used as the .repeater-start element's text value.

      Plugins

      The repeater has a view framework that can be extended with plugins to represent data in different ways. By default, Fuel UX comes with two repeater view plugins (list and thumbnail), each providing additional options and requiring different return data from the dataSource to render appropriately. For information on using these plugins, visit the plugins page. Additionally, learn how to write your own repeater plugin here.
